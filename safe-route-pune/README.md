# SafeRoute Pune 🛡️
### AI-powered safe route navigation for Pune, Maharashtra

> Analyses crime hotspots, accident blackspots, street-lighting coverage and real-time crowd-sourced reports to help you choose the safest route across Pune.

## Using the app

### Planning a route

1. Type your **starting point** (e.g. "Baner, Pune") in the top search box.
2. Type your **destination** (e.g. "Hadapsar, Pune") in the second box.
3. Click **"Find safe routes"**.
4. Three route cards appear:
   - ⚡ **Fastest** — shortest travel time (Google Maps default)
   - 🛡️ **Safest** — highest safety score from our model
   - ⚖️ **Balanced** — weighted blend of time and safety
5. Each card shows a **safety score out of 100** and a colour-coded bar.
6. Select a mode pill at the top to switch which route is shown on the map.
7. Click **"Start journey"** to begin.

### Safety score meaning

| Score | Label | Colour |
|-------|-------|--------|
| 70–100 | Safe | 🟢 Green |
| 50–69 | Moderate | 🟡 Amber |
| 30–49 | Risky | 🟠 Orange |
| 0–29 | Dangerous | 🔴 Red |

### Map layers (toggles in the sidebar)

| Layer | Shows |
|-------|-------|
| Safety heatmap | Red/green density overlay from our model |
| Accident hotspots | Circles at Pune's known accident blackspots |
| Safety zone info | Numbered markers with detailed zone breakdown |
| Women safety areas | Purple markers for women-specific risk zones |

### Submitting feedback (authenticated users only)

After clicking **"End journey"** a feedback modal appears:

1. Rate **overall experience** (1–5 stars)
2. Rate **how safe the route felt** (1–5 stars)
3. Optionally add a comment
4. If you felt unsafe somewhere, tick the checkbox → click the map to pin the location → describe what happened
5. Click **"Submit feedback"**

Your report immediately adjusts the safety score for that area and is reflected
in the heatmap for all users.

## Safety dataset details

The dataset covers the **Pune Municipal Corporation** and **PCMC** areas
(bounding box 18.39°N–18.66°N, 73.70°E–73.98°E).

### Risk zones included

| Risk level | Example areas |
|-----------|---------------|
| CRITICAL | Katraj Ghat (accident blackspot) |
| HIGH | Yerwada, Bhosari MIDC, Kondhwa, Dhanori, Wagholi |
| MODERATE | Hadapsar, Koregaon Park, Camp, Deccan, Pimpri |
| LOW/SAFE | Aundh, Baner, Kothrud, Magarpatta, Hinjewadi, Balewadi |

### Score components (0–100 each)

- `crime` — IPC offences per zone (NCRB 2022-23)
- `accidents` — vehicle accidents per zone (Maharashtra Traffic Police 2022-23)
- `lighting` — PMC street-light density audit 2023
- `women_safety` — WCD Safe City Project Pune 2023 incidents

**Overall** = weighted average of the four components.

The route score = 70% average + 30% minimum segment score
(penalises routes that pass through even one dangerous stretch).

## How crowdsourcing updates the model

1. User completes a journey and marks a location as unsafe.
2. The server writes a **safety adjustment record** (lat, lng, negative delta).
3. On the next heatmap request the server applies inverse-distance weighting
   of all adjustments to the base heatmap points.
4. Positive feedback (safety ≥ 4 stars) adds a small positive adjustment at
   the route midpoint.
5. Over time, areas with consistent negative reports move from green → amber
   → red on the heatmap; well-reported safe corridors stay green.

To retrain the full ML model after significant crowd feedback:
```bash
npm run train
```

## Scaling beyond Pune

To add another city:

1. Add zone definitions in `ml/generate_dataset.py` (copy the `PUNE_SAFETY_ZONES` pattern).
2. Update `PUNE_BOUNDS` and `PUNE_CENTER` constants.
3. Run `npm run generate-data` and `npm run train`.
4. Update `PUNE_CENTER` in `public/dashboard.html` (line with `const PUNE_CENTER`).

## Common issues

| Issue | Fix |
|-------|-----|
| `Safety data not found` | Run `npm run generate-data` first |
| Map shows blank / API error | Check Google Maps API key restrictions; enable Maps JS, Places, Directions APIs |
| `Cannot find module 'lowdb'` | Run `npm install` |
| Port 3000 in use | Change `PORT=` in `.env` |
| Login always fails | Delete `data/db.json` and restart (re-seeds demo user) |
| Python script fails | Run `pip3 install scikit-learn pandas numpy joblib --break-system-packages` |
